# Task Manager Sync Information

## What Syncs to Child Projects

The task management system is designed to sync utilities and tools to child projects, but NOT the actual task list.

### ✅ Files That SYNC (Template → Child Projects)

All task management utilities and documentation sync automatically:

**CLI Tools:**
- `task-manager/tasks-cli.ts` - CLI implementation
- Scripts in `package.json` (`task`, `task:list`, `task:work`, etc.)

**Documentation:**
- `task-manager/README.md` - Overview
- `task-manager/TASK_COMMANDS.md` - Quick reference
- `task-manager/SLASH_COMMANDS_README.md` - Complete guide
- `task-manager/task-management-cli.md` - CLI documentation

**Slash Commands:**
- `.claude/commands/add-task.md` - `/add-task` command
- `.claude/commands/task-list.md` - `/task-list` command
- `.claude/commands/start-task.md` - `/start-task` command
- `.claude/commands/start-task-worktree.md` - `/start-task-worktree` command

**Related Documentation:**
- `docs/git-worktree-workflow.md` - Worktree workflow guide

### ❌ Files That DO NOT SYNC (Project-Specific)

**Task List:**
- `task-manager/tasks.md` - Each project has its own tasks

**Configuration:**
- Listed in `.template-sync.json` under `projectSpecificFiles`

## Why This Design?

### Tools Sync
Child projects benefit from:
- Latest CLI improvements
- Updated slash commands
- Better documentation
- Bug fixes in task management tools

### Tasks Don't Sync
Each project has different:
- Task priorities
- Implementation details
- Project-specific requirements
- Progress tracking

## For Child Projects

When you sync from template:
1. ✅ You GET the latest task management tools
2. ✅ You GET updated documentation
3. ✅ You GET new slash commands
4. ❌ You KEEP your own `tasks.md` (never overwritten)

## Initial Setup in Child Projects

When creating a new child project from this template:

1. **First Time Only:** Create your own `task-manager/tasks.md`:
   ```bash
   # In child project
   mkdir -p task-manager
   cat > task-manager/tasks.md << 'EOF'
   # Tasks

   Tasks sorted by priority (Critical → High → Medium → Low).

   ---

   ## 1. Example Task

   | Priority | Complexity | Size |
   |----------|------------|------|
   | **High** | Low | S |

   **Summary:** Example task description

   **Files to Modify:**
   - `path/to/file.ts` - What to change
   EOF
   ```

2. **All Other Times:** Run `yarn sync-template` to get latest tools
   - Your `tasks.md` is protected and won't be overwritten

## Configuration

See `.template-sync.json`:

```json
{
  "projectSpecificFiles": [
    "task-manager/tasks.md"
  ]
}
```

This ensures `tasks.md` is never synced from template to child projects.
